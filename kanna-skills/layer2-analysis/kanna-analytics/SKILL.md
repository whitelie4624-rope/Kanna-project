# KANNA アクセス解析Skill

## 概要
KANNAのWebサイトアクセスデータをKANNA視点で解釈し、改善アクションを提案するSkillです。ユーザーがデータを貼り付けると即座に解釈します。女風特有の検索行動・顧客心理を考慮した分析を行います。

## KANNAについて
- **サービス種別**：女性用風俗（女風）・リラクゼーション
- **ターゲット**：20〜40代女性
- **場所**：東京
- **SNS**：Instagram・X（旧Twitter）・note
- **重視すること**：リピート率・顧客との信頼関係・ブランド一貫性

## 手順

### Step 1：データの受け取り
ユーザーがGoogle Analytics・Search Console等のデータを貼り付けたら即座に解釈を開始する。
以下のデータがあれば分析精度が上がる：
- セッション数・ユーザー数（期間比較）
- 流入経路（オーガニック / SNS / 直接 / 参照）
- よく見られているページ
- 直帰率・滞在時間
- 検索クエリ（Search Consoleデータ）
- CVR（予約ページへの到達率・予約完了率）

### Step 2：女風特有の検索行動の考慮
以下の特性を踏まえて分析する：
- 女性が女風を検索する際は「リラクゼーション」「癒し」「女性向け」などの
  間接的なキーワードを使う傾向がある
- SNS経由の流入は「口コミ・共感」ベースであり、直接予約に繋がりやすい
- 初回来店前に複数回サイトを閲覧する傾向がある（信頼確認の行動）
- 夜間・昼休み・週末にアクセスが集中しやすい

### Step 3：各指標の解釈
**セッション数：**
- 絶対数より前期比・前月比で評価する
- SNSフォロワー数との相関を確認する

**流入経路：**
- Instagram / X / note からの流入比率を確認
- オーガニック検索のキーワードを確認してSEO対策の優先度を判断

**直帰率・滞在時間：**
- トップページの直帰率が70%を超える場合は改善余地あり
- 予約ページへの誘導ボタンの配置・文言を確認する

**CVR（予約率）：**
- 予約ページ到達 → 予約完了の転換率を計算する
- 離脱ポイントを特定する

### Step 4：改善アクションの優先度付け
「影響が大きい × 実行が容易」の順に改善アクションを提案する

## 注意事項
- データが不完全な場合は、分析できる範囲を明示して進める
- 数値の解釈は「良い / 普通 / 要改善」の3段階で評価する
- 改善提案は「一人運営で実行可能なもの」を優先する
- 顧客の個人情報（IPアドレス等）には言及しない
- Google Analyticsのバージョン（UA / GA4）によって指標の定義が異なることを考慮する

## 出力例

```
# KANNAアクセス解析レポート（2024年4月）

## 解釈コメント
### 良い傾向
- Instagram経由の流入が前月比+35%。投稿頻度増加の効果が出ている
- 予約ページの滞在時間が平均3分以上：料金・コース説明への関心度が高い

### 懸念点
- トップページ直帰率：72%（改善目標：60%以下）
  → ファーストビューに「初めての方へ」の導線を追加することを推奨

## 改善アクションリスト
### 優先度：高
1. トップページに「はじめての方へ」ボタンを追加 → 直帰率改善・CVR向上見込み
```
