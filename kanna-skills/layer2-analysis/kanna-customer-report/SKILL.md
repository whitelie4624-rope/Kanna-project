# KANNA 顧客分析レポートSkill

## 概要
KANNAの顧客全体の傾向・属性・来店パターンを分析してレポート化するSkillです。ユーザーが入力した顧客データをもとに、CRM施策につながる示唆を生成します。

## KANNAについて
- **サービス種別**：女性用風俗（女風）・リラクゼーション
- **ターゲット**：20〜40代女性
- **場所**：東京
- **重視すること**：リピート率・顧客との信頼関係・ブランド一貫性

## 手順

### Step 1：顧客データの受け取り
以下のデータを入力してもらう（個人を特定する情報は省いてよい）：
- 顧客総数
- 年齢層の分布（20代 / 30代 / 40代 / その他）
- 来店頻度の分布（月1回 / 2ヶ月に1回 / 3ヶ月以上 / 1回限り）
- 予約経路の分布（Instagram / X / 予約サイト / 口コミ / その他）
- よく選ばれるプラン
- 来店時間帯の傾向（平日昼 / 平日夜 / 休日）

データが少ない場合は、セラピストの肌感覚・記憶でもOK

### Step 2：属性分析
- 主力顧客層（最も多い属性）の特定
- サブ顧客層の特定
- 期待していた想定ターゲットとの乖離確認

### Step 3：来店パターン分析
- 来店頻度のパターン（高頻度 / 中頻度 / 低頻度）別の人数割合
- 来店間隔の傾向（平均何日ごとに来店するか）
- リピーターの来店サイクルの特定

### Step 4：予約経路分析
- 最も効果的な集客チャネルの特定
- 各チャネルの新規顧客獲得力とリピート転換率の比較

### Step 5：CRM施策の提案
顧客分析をもとに、以下のカテゴリで施策を提案する：
- **リピート促進**：コア顧客をさらに定着させる施策
- **失客防止**：来店頻度が落ちた顧客へのフォロー
- **新規獲得**：不足している顧客層を補う施策

## 注意事項
- 個人を特定できる情報（氏名・連絡先・詳細な年齢）はレポートに含めない
- 顧客データは経営改善目的のみで使用し、外部共有しない
- データが少ない場合は「サンプル数が少ないため参考値」と明記する
- 分析結果は「傾向」として提示し、断定的な表現は避ける
- 顧客の来店理由や悩みに関する情報は、プライバシーを尊重して扱う

## 出力例

```
# KANNA 顧客分析レポート（2024年1〜3月）

## 顧客プロフィールサマリー
- 顧客総数：28名
- 主力年齢層：30代（52%）
- 主力来店頻度：月1回前後
- 主力予約経路：Instagram（61%）

## 顧客インサイト
1. 30代会社員・フリーランス層が主力。「仕事のストレス発散」「自己へのご褒美」が
   主な来店動機と推測される
2. Instagram経由の新規が多い。写真・世界観への共感で来店を決める傾向が強い
3. コア顧客（月1回以上）が全体の35%を占め、収益の60%以上を支えている

## CRM施策提案
### 失客防止
- 2ヶ月来店のない顧客に自然な気遣いDMを送付（kanna-lost-alertと連携）
- 「久しぶりに会いたいな」という温かみのある文体で接触する
```
